# 智汇教材 — AI 驱动的多源教材知识整合平台

## 一句话简介

让教师用自然语言整理、对比、融合多本教材，自动构建学科知识图谱，生成压缩至 30% 的教学精华。

---

## 问题背景

一线教师和教学管理者在日常工作中反复面临三个痛点：

1. **教材碎片化**：同一学科有多本教材，知识点分散在数百页的 PDF、Word、Excel 材料中，人工比对耗时数周。
2. **知识整合无标准**：跨教材的概念重复、互补、缺失和冲突缺乏结构化发现手段，凭经验判断容易遗漏关键差异。
3. **教学精华缺失**：要从海量原始教材中提炼一份"最该教什么、按什么顺序教"的浓缩教案，现有工具帮不上忙。

AI 能读书、能理解、能对话，但现有 AI 工具只能陪你聊天——它们无法**同时啃下多本教材**、**可视化知识点之间的关系**、**接受教师的修改意见并重建**、**生成量化的教学精华**。

---

## 产品简介

**智汇教材（TextBook Refiner）** 是一个面向教师的教学知识整合智能体。教师将多本教材拖入平台，系统在**没有预置知识库**的前提下实时完成：

- 自动解析 PDF / Markdown / Word / Excel 四种格式
- 提取每个教材的章节结构，按章节构建知识图谱
- 识别知识点之间的关系：前置依赖、并列教学、包含关系、应用关系等
- 跨教材比对，发现**重复知识点、互补知识点、缺失知识点、冲突逻辑链**
- 根据前置关系生成最优教学顺序，覆盖所有关键概念
- 生成压缩至原文字数 30% 的**教学精华版**，教师可导出为报告

**区别于现有 AI 教育工具的核心差异**：这不是一个"AI 聊天问答"应用。它是一个**可交互的知识整合工作台**——教师能看到图谱动态生长、能参与决策、能调整参数让系统重建、能导出教学报告。

## 核心功能模块

### 1. 多格式教材导入

| 格式 | 说明 |
|------|------|
| PDF | 医学教材等高密度文档，支持章节自动识别 |
| Markdown | 技术文档、教案等结构化文本 |
| Word (.docx) | 教学大纲、讲义 |
| Excel (.xlsx) | 列表型知识点、术语表 |

上传即解析，无需等待——每本教材上传后立即完成格式识别、文本提取和章节划分。

### 2. 动态知识图谱构建

系统从教师上传的教材中**实时抽取知识点和关系**，不对原始教材做预制处理：

- **节点类型**：教材 → 章节 → 知识点（三层树状结构 + 知识点间关系网络）
- **关系类型**：前置依赖、并列教学、包含关系、应用关系、致因关系、归属关系、临床表现为、解剖定位于、一般关联
- **粒度控制**：每个知识点定义粒度（GRAIN 规则）为"可独立讲解的最小教学单元，1-3 句能讲清"
- **可视化呈现**：使用力导向图动态渲染，教师可缩放、拖拽、点击查看知识点定义和出处

### 3. 跨教材知识整合

当多本教材的图谱构建完成后，系统自动执行四类分析：

| 类型 | 说明 | 示例 |
|------|------|------|
| **重复** | 两本教材中实质相同的知识点 | "炎症定义"在病理学、病理生理学中各出现一次 |
| **互补** | 同一主题在不同教材中有不同侧重点 | 传染病学讲"免疫应答"侧重抗体，微生学讲侧重细胞免疫 |
| **缺失** | 前置关系链中明显缺少中间概念 | "血管收缩" → "血压升高"之间缺少"外周阻力"概念 |
| **冲突** | 两本教材对同一概念有矛盾表述或相反的前置依赖 | A 书说 X→Y，B 书说 Y→X |

每条整合建议附带：
- **置信度评分**（0~1）
- **原文证据引用**（教材名、章节、页码范围、原文摘录）
- **建议的图谱操作**（合并节点 / 新建关系 / 拆分概念 / 更新定义）

### 4. 教师对话工作台

教师在聊天面板中用自然语言发出指令，系统调用图谱工具：

- "把'炎症反应'和'炎症'合并" → 调用 `merge_nodes`
- "病毒性肝炎应该在免疫应答之前讲" → 调用 `add_edge`（prerequisite）
- "这个定义不对，应该是……" → 调用 `update_definition`
- "帮我查一下病理学怎么讲心力衰竭的" → 调用 RAG 检索 + 带引用回答

**所有工具调用都是结构化、可追溯的**——教师能看到系统做了什么操作、基于什么理由、影响了哪些节点和边。

### 5. 教学精华生成

基于完成整合的知识图谱，系统自动：

1. **生成教学顺序**：拓扑排序尊重所有前置依赖，标记无法解决的循环依赖供教师确认
2. **生成压缩精华**：LLM 根据图谱节点和关系生成连贯的教学精华文本，目标 ≤ 原始字符数 × 30%
3. **量化压缩报告**：原始字数、精华字数、压缩比、决策汇总——全部可导出

### 技术选型理由

| 层级 | 选型 | 理由 |
|------|------|------|
| 前端框架 | Next.js 16 + React 19 | App Router 路由清晰、SSR 友好、TypeScript 严格模式 |
| UI 组件库 | shadcn/ui (基于 @base-ui/react) | 源码可控、Tailwind v4 原生支持、零运行时成本 |
| 图谱可视化 | react-force-graph-2d (Canvas) | 力导向图开箱即用、Canvas 渲染性能优于 SVG 方案、支持自定义节点绘制 |
| 后端框架 | FastAPI + async | 高并发文档解析、WebSocket 原生支持、Pydantic 类型安全 |
| 图数据库 | NetworkX + JSON | 无需额外部署 Neo4j、操作日志可审计、状态变更可回溯 |
| 向量数据库 | ChromaDB | 部署极简（单容器）、支持 HTTP 和持久化两种模式、cosine 距离原生支持 |
| 模型接入 | DeepSeek v4-pro + ModelScope Qwen3-Embedding-8B | 国内可用、OpenAI 兼容 API、推理 + 嵌入各司其职 |

```

                    │         │
```

## 核心竞争力

### 1. 真实可交互，而非"AI 聊天问答"

大部分 AI 教育应用是对话式——你问，它答。智汇教材是一个**可视化工作台**：教师能看到知识图谱从零生长的全过程，能点击查看每个知识点的定义、出处、关联关系，能用自然语言"操作"图谱而不是"问"图谱。

### 2. 治理颗粒度可达知识点级别

GRAIN 规则定义了什么是"一个知识点"，DUPLICATE 规则定义了如何判断两个知识点是重复的。这些规则**不是写死在代码里的**——它们通过 LLM 提示词约束和结构化 schema 保证输出质量，教师可通过对话随时修正。

### 3. 所有 AI 决策附带证据链

每条整合建议、每个关系提取、每个教学顺序决定都附带**教材名 + 章节 + 页码范围 + 原文摘录**。教师不信任 AI 的判断时，可以直接溯源到原书内容。

### 4. 教师反馈闭环

教师的自然语言修正会转换为**结构化图操作**，系统理解、执行、保存，并推送到前端重新渲染。没有"先提交再等结果"的延迟——图谱状态实时更新。

### 5. 量化交付物

不是"AI 帮你整理了一下"，而是**精确到字符数的教学精华**：
- 原始总字数：248,356 字
- 精华字数：71,203 字
- 压缩比：28.7%（≤ 30% 达标）
- 教学顺序：87 个步骤，前置依赖全部满足

## 使用场景

### 场景一：教师备课

张老师在准备"病理学"课程，手上有 5 本不同年代的教材。她将所有 PDF 拖入智汇教材，系统用 5 分钟构建了完整的跨教材知识图谱。她浏览整合决策队列——发现有 32 个重复概念可以合并、8 个互补概念可以关联。她接受大部分建议后，系统生成了整合后的教学顺序和压缩精华。她导出报告发给教研室审核。

### 场景二：教材评审

李教授在评审三本新编医学教材。他将教材上传到智汇教材，系统识别出三本教材在"免疫学基础"部分有显著差异——A 书放置在第一章，B 书放置在第五章，C 书分散在三章中。系统标记了 12 处潜在的"缺失知识点"，李教授逐一确认后，生成了一份结构化的教材对比评审报告。

### 场景三：教学改革研讨

教研室主任王老师想在课程改革中整合多学科内容。她组织了来自病理学、生理学、微生物学的 7 本教材，在智汇教材中构建了跨学科知识图谱。系统发现了 45 个跨学科的互补知识关联和 3 个前置依赖冲突。教研组在聊天面板中逐一讨论、修正，最终产出了新的跨学科教学大纲。

## 即刻开始

```bash
# 1. 配置 API 密钥
cp .env.example .env
# 编辑 .env 填入 DeepSeek API Key 和 ModelScope API Key

# 2. 启动服务
docker compose up -d

# 3. 打开浏览器
open http://localhost:3000
```

**浏览器中操作流程**：创建项目 → 拖入教材 → 点击"开始构建" → 查看图谱生长 → 审阅整合建议 → 与教师智能体对话 → 生成教学精华报告
